Overview

Questify is a modern, lightweight Computer Based Testing (CBT) system built entirely with HTML5, CSS3 (Tailwind), and vanilla JavaScript. It provides a complete examination workflow from student login to certificate generation, with no backend dependencies required.

What's Included

  • 6 fully responsive pages with modern UI design
  • Session management with 30-minute auto-expiry
  • Live camera photo capture with fallback file upload
  • Professional certificate generation with print functionality
  • Exam timer with auto-submit on timeout
  • Question navigator showing answered/unanswered status

Technical Specifications

Frontend
  • • HTML5 semantic markup
  • • Tailwind CSS v3.x via CDN
  • • Vanilla JavaScript (ES6+)
  • • No jQuery or frameworks
Storage & Dependencies
  • • SessionStorage for session data
  • • Base64 encoded photo storage
  • • Requires internet (Tailwind CDN)
  • • No backend required

Step 4: Understanding the CDN Dependency

This template uses Tailwind CSS via CDN for styling. This means:

✓ Advantages:

  • • Zero build process required
  • • Works immediately out of the box
  • • No Node.js or npm installation needed
  • • Perfect for quick deployment and testing

⚠ Requirements:

  • • Internet connection required on first page load
  • • CDN loads from: https://cdn.tailwindcss.com
  • • After initial load, browser caching makes subsequent loads fast

Note: For production environments with strict offline requirements, you can compile Tailwind CSS locally and replace the CDN script with a stylesheet link. See the Customization section for details.

Customization Guide

1 Changing Login Credentials

Edit the validCredentials array in js/index.js

// In js/index.js
const validCredentials = [
{
examNumber: 'EXM2025001',
accessCode: 'DEMO2025',
studentName: 'John Doe',
examTitle: 'General Knowledge Test'
},
// Add more credentials here
];

2 Modifying Exam Questions

Edit the questions array in js/exam.js

// In js/exam.js
const questions = [
{
question: "Your question here?",
options: ["Option A", "Option B", "Option C", "Option D"],
correctAnswer: 0 // Index (0 = first option)
},
// Add more questions...
];

Note: correctAnswer is zero-indexed (0 = first option, 1 = second option, etc.)

Limit: The question navigator uses a 5-column grid. For more than 50 questions, you'll need to adjust the grid layout in exam.html or add pagination.

3 Changing Exam Duration

Modify timer in js/exam.js

// In js/exam.js
let timeLeft = 45 * 60; // 45 minutes in seconds
// Examples:
let timeLeft = 30 * 60; // 30 minutes
let timeLeft = 60 * 60; // 60 minutes
let timeLeft = 90 * 60; // 90 minutes

4 Changing Colors/Branding

The design uses orange as the primary color. To change colors, update Tailwind classes:

Find & Replace in all HTML files:

orange- → blue- (for blue theme)
orange- → green- (for green theme)
orange- → purple- (for purple theme)

5 Changing Session Expiry Time

Default session is 30 minutes. Modify in js/index.js

// In js/index.js - login handler
expiresAt: new Date(Date.now() + 30 * 60 * 1000)
// Change 30 to desired minutes:
expiresAt: new Date(Date.now() + 60 * 60 * 1000) // 60 minutes

6 Bypassing Exam Countdown (Testing)

For testing, set exam start time to the past in js/verify-credentials.js

// In js/verify-credentials.js
// FOR PRODUCTION (5 minutes wait):
const EXAM_START_TIME = new Date(Date.now() + 5 * 60 * 1000)
// FOR TESTING (instant access):
const EXAM_START_TIME = new Date(Date.now() - 1000)

Warning: Remember to change this back for production!

7 Converting CDN to Local CSS (Advanced)

If you need offline functionality or want to optimize file size, you can replace the CDN with a local CSS file.

Current Setup (CDN):

<script src="https://cdn.tailwindcss.com"></script>

Option 1: Use Full Tailwind CSS

  1. 1. Download full CSS from: https://unpkg.com/tailwindcss@3.4.17/dist/tailwind.min.css
  2. 2. Save as css/tailwind.min.css
  3. 3. Replace script tag with: <link rel="stylesheet" href="./css/tailwind.min.css">

Option 2: Compile Tailwind Locally (Recommended for Production)

  1. 1. Install Node.js from nodejs.org
  2. 2. Run: npm install -D tailwindcss
  3. 3. Create config file and compile (see Tailwind docs for full guide)
  4. 4. Replace CDN script with compiled CSS link

For most users: The CDN approach works perfectly and requires no additional setup. Only convert to local CSS if you have specific offline requirements or want to optimize file size further.

Session Management

Questify uses browser SessionStorage to manage user authentication and exam state. Sessions automatically expire after 30 minutes for security.

How It Works

  1. 1. User logs in with valid credentials → session created in sessionStorage
  2. 2. Session includes: examNumber, studentName, examTitle, loginTime, expiresAt
  3. 3. Every page checks session validity on load
  4. 4. If expired or missing → redirect to login
  5. 5. Session cleared on logout or browser close
// Session data structure
{
examNumber: "EXM2025001",
studentName: "John Doe",
examTitle: "General Knowledge Test",
loginTime: "2025-10-03T14:30:00.000Z",
expiresAt: "2025-10-03T15:00:00.000Z",
photoVerified: true,
photoDataUrl: "data:image/jpeg;base64,..."
}

Security Features

  • • Auto-expiry prevents unauthorized access
  • • Session cleared on browser close
  • • Each page validates session before rendering
  • • No persistent storage (no localStorage used)
  • • Session lost if browser refreshed during exam - answers not saved

Browser Storage

The system uses sessionStorage exclusively. No localStorage, cookies, or server-side storage required.

What We Use

✓ sessionStorage - Temporary data during exam session
✓ Auto-clears when browser closes
✓ No backend database needed

What We DON'T Use

✗ localStorage - Not used (would persist after session)
✗ Cookies - Not needed
✗ Backend database - Fully client-side

Important for Integration

When integrating with your backend, you can easily replace sessionStorage calls with API requests to save data permanently. The current setup demonstrates the complete workflow without requiring server infrastructure.

Frequently Asked Questions

Do I need a backend/database?

No! This is a fully client-side system. It works perfectly without any backend. However, you can easily integrate it with your own backend by replacing sessionStorage calls with API requests.

Can I use this commercially?

Yes! This template comes with a Regular License for personal and commercial use. You can use it for client projects, your own business, or educational institutions.

How do I add more questions?

Open js/exam.js and add more objects to the questions array. Follow the existing format. Don't forget to update the total question count in the UI.

Is the camera feature required?

No. The system automatically falls back to file upload if camera access is denied or unavailable. Users can upload a photo from their device instead.

Can I change the design/colors?

Yes! The design uses Tailwind CSS classes. Simply find and replace color classes (e.g., change orange- to blue-) throughout the HTML files.

Does it work offline?

Almost. The system uses Tailwind CSS from CDN, which requires internet. To make it fully offline, download Tailwind CSS locally and update the stylesheet links.

Can I skip the photo verification step?

Yes. Simply modify the navigation flow. In verify-credentials.html, change the "Proceed" button to go directly to student-dashboard.html instead of upload-photo.html.
